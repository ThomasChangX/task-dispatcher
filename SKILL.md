## 触发条件

**任何用户任务**触发，无需额外关键词：
- 用户提交任何类型的任务（修复、新功能、调研、文档等）
- 接收到需要多步骤完成的工作

## 6 阶段核心流程

```
用户任务 → [需求分析] → [研究分析] → [方案确认] → [任务拆解] → [执行验证] → [收尾]
                    ↓           ↓           ↓            ↓            ↓
               提取目标    信息收集    等用户确认   分发Agent    验证质量    归档记录
```

### 阶段说明

| 阶段 | 核心动作 | 产出 |
|------|----------|------|
| 1. 需求分析 | 提取任务目标、约束条件、判定任务类型 | 任务分析报告 |
| 2. 研究分析 | 调用 researcher 收集相关信息（必须） | 研究报告 |
| 3. 方案确认 | 基于研究产出制定方案，等待用户确认 | 确认方案 |
| 4. 任务拆解 | 生成结构化任务列表 | 任务表格 |
| 5. 执行验证 | 分发 sub-agent 执行，reviewer/tester 验证 | 执行结果 |
| 6. 收尾 | 文档归档、记忆更新 | 完成报告 |

## 关键原则

- **必须研究分析**：不可跳过研究阶段，无研究不执行
- **先确认后执行**：任务列表必须用户确认才能分发
- **信息不足就问**：不猜测，主动向用户确认模糊点
- **透明进度**：及时汇报，不让用户猜测状态
- **兜底记录**：每次任务完成后记录到 memory/YYYY-MM-DD.md
- **全走 dispatcher**：不区分简单复杂，所有任务统一处理

## 差异化定位

与现有 task-dispatcher (v1) 的区别：
- **更严格的流程**：6 阶段缺一不可
- **强制研究分析**：研究阶段不可跳过
- **明确的阶段产出**：每个阶段有明确的交付物
- **更细的颗粒度**：详细的判定规则和决策点

## 快速入门

### 首次使用

```bash
# 初始化会话环境（可选，首次使用建议运行）
python3 ~/.openclaw/workspace/skills/task-dispatcher/scripts/init_session.py
```

### 工作流程

1. **接收任务** → 自动进入需求分析
2. **等待确认** → 方案确认后等待你确认
3. **执行交付** → 分发 agent 执行并汇报结果

### 典型交互

```
你: 帮我修复登录页面的 bug
↓
[需求分析] 提取任务目标：修复登录页面 bug
↓
[研究分析] 调用 researcher 收集信息
↓
[方案确认] 展示方案，等待你确认
↓
[任务拆解] 展示任务表格，等待你确认
↓
[执行验证] 分发 agent 执行
↓
[收尾] 汇报完成，记录到记忆系统
```

## v1 vs v2 如何选择

| 场景 | 推荐版本 |
|------|----------|
| 需要严格流程保证质量 | **v2 (本 skill)** |
| 简单任务，追求效率 | v1 (task-dispatcher) |
| 复杂/重要任务 | **v2 (本 skill)** |
| 快速调研/查询 | v1 (task-dispatcher) |
| 需要明确的阶段产出 | **v2 (本 skill)** |

**选择原则**：不确定时用 v2，更严格的流程能减少返工。

## 使用方式

1. 用户提交任务
2. 自动进入 6 阶段流程
3. 每个关键节点向用户汇报
4. 执行完成后自动记录到记忆系统

### Session 初始化（可选）

```bash
# 每次新任务可以初始化会话环境
python3 ~/.openclaw/workspace/skills/task-dispatcher/scripts/init_session.py
```

这会创建/更新 `SESSION-STATE.md`，记录当前任务状态。
