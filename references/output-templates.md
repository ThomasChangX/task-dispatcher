# 输出格式模板

本文件定义了 6 阶段流程中各阶段的标准输出格式。

---

## 1. 任务分析报告

### 用途
需求分析阶段产出

### 模板

```markdown
## 任务分析报告

### 任务来源
[用户/系统/定时]

### 目标
[用户想要达成什么]

### 约束条件
- 时间约束: [如有]
- 技术约束: [如有]
- 预算约束: [如有]

### 任务类型
[修复/新功能/调研/文档/架构设计/部署运维/其他]

### 复杂度
[简单/一般/复杂/超大]

### 初步判断
[基于当前信息的初步判断]
```

### 示例

```markdown
## 任务分析报告

### 任务来源
用户

### 目标
修复登录页面点击登录按钮无反应的问题

### 约束条件
- 时间约束: 尽快
- 技术约束: 现有 React 框架

### 任务类型
修复

### 复杂度
一般

### 初步判断
需要先通过 inspector 定位问题，可能是前端事件绑定或 API 调用问题
```

## 2. 研究报告

### 用途
研究分析阶段产出

### 模板

```markdown
## 研究报告

### 研究目标
[研究要回答的问题]

### 背景
[相关背景信息]

### 发现
#### 发现 1
[详细内容]

#### 发现 2
[详细内容]

### 建议方案
[基于研究的建议]

### 风险点
- 风险 1: [描述]
- 风险 2: [描述]

### 参考资料
- [资料1]
- [资料2]
```

### 示例

```markdown
## 研究报告

### 研究目标
确定登录无反应的根本原因

### 背景
用户反馈点击登录按钮后页面无任何反应

### 发现
#### 发现 1
检查 login.tsx，发现 onClick 事件绑定到了 handleLogin 函数

#### 发现 2
handleLogin 函数内部调用了 /api/auth/login，但该 API 返回 404

### 建议方案
修复 API 路由或修改前端调用地址

### 风险点
- 可能影响其他调用该 API 的模块

### 参考资料
- src/pages/login.tsx
- server/routes/auth.js
```

## 4. 任务拆解表格

### 用途
任务拆解阶段产出

### 模板

```markdown
## 任务拆解 - {任务名称}

### 执行计划

| 序号 | 任务 | Agent | must_haves | 依赖 | 状态 |
|------|------|-------|------------|------|------|
| 1 | xxx | xxx | - | - | 待执行 |
| 2 | xxx | xxx | 验证点 | 1 | 待执行 |
| 3 | xxx | xxx | 审查标准 | 2 | 待执行 |

### 列说明
| 列 | 说明 |
|----|------|
| 序号 | 任务编号 |
| 任务 | 具体执行内容 |
| Agent | 负责执行的 agent |
| must_haves | 可验证的硬性交付标准，来自研究分析阶段 |
| 依赖 | 前置任务序号，无依赖用 `-` |
| 状态 | 待执行 / 执行中 / 已完成 / 失败 |

### 串行任务
- [序号] 必须在前置任务完成后才能开始

### 并行任务
- [序号A] 和 [序号B] 可并行执行

### 评审节点
- [阶段]: [评审者]

---

**请确认任务拆解，确认后立即执行。**
```

### 示例

```markdown
## 任务拆解 - 登录问题修复

### 执行计划

| 序号 | 任务 | Agent | must_haves | 依赖 | 状态 |
|------|------|-------|------------|------|------|
| 1 | 定位登录无反应根因 | inspector | 定位报告包含根因分析 | - | 待执行 |
| 2 | 修复 API 路由问题 | coder | 登录功能恢复正常 | 1 | 待执行 |
| 3 | 验证修复效果 | tester | 登录功能测试通过 | 2 | 待执行 |
| 4 | 代码审查 | reviewer | 代码符合规范 | 2 | 待执行 |

### 串行任务
- 2 依赖 1 完成
- 3、4 依赖 2 完成

### 评审节点
- 修复后: reviewer 代码审查
- 验证后: tester 功能验证

---

**请确认任务拆解，确认后立即执行。**
```

## 5. 阶段汇报

### 用途
执行过程中的进度汇报

### 模板

```markdown
## 阶段汇报 - {任务名称}

### 当前阶段
[阶段名称]

### 进度
- 已完成: [X/Y] 个子任务
- 进度: [XX%]

### 最新动态
- [动态1]
- [动态2]

### 等待事项
- [如有等待事项]

### 预计
- [预计完成时间]
```

## 6. 任务完成报告

### 用途
任务完成后的最终汇报

### 模板

```markdown
## 任务完成 - {任务名称}

### 状态
✅ 完成 / ❌ 失败 / ⚠️ 部分完成

### 完成时间
[开始时间] → [结束时间]

### 产出
- [产出1]
- [产出2]

### 执行记录
| 步骤 | Agent | 结果 |
|------|-------|------|
| 1 | xxx | ✅ |
| 2 | xxx | ✅ |
| 3 | xxx | ✅ |

### 经验教训
- [如有]

### 下一步建议
- [如有]
```

```markdown
[问题描述]

### 尝试的处理
- [尝试1]
- [尝试2]

### 当前状态
[状态描述]

### 待处理
- [待处理事项]
```

|------|----------|----------|
| 需求分析 | 任务分析报告 | 目标、类型、复杂度 |
| 研究分析 | 研究报告 | 发现、建议、风险 |
| 方案确认 | 方案确认 | 方案要点、等待确认 |
| 任务拆解 | 任务表格 | 子任务、依赖、agent |

| 完成 | 完成报告 | 产出、记录 |
| 异常 | 兜底记录 | 问题、待处理 |
