# 任务表格模板

## 基础模板

| 序号 | 任务 | Agent | 依赖 | 状态 | 备注 |
|------|------|-------|------|------|------|
| 1 | 任务描述 | agent名 | - | 待执行 | 备注信息 |

## 模板说明

### 字段说明

| 字段 | 说明 | 示例 |
|------|------|------|
| 序号 | 任务唯一编号 | 1, 2, 3 |
| 任务 | 任务描述 | 修复登录问题 |
| Agent | 执行的 agent | coder, tester |
| 依赖 | 依赖的任务序号 | 1, 2 |
| 状态 | 当前状态 | 待执行/进行中/已完成/失败 |
| 备注 | 额外说明 | 优先级高 |

### 状态值

- `待执行` - 等待执行
- `进行中` - 正在执行
- `已完成` - 执行成功
- `失败` - 执行失败
- `已跳过` - 因故跳过

### 依赖表示

- `-` 或空 - 无依赖，可立即执行
- `1` - 依赖任务1
- `1,2` - 依赖任务1和2

## 并行任务示例

当多个任务可并行执行时：

| 序号 | 任务 | Agent | 依赖 | 状态 | 备注 |
|------|------|-------|------|------|------|
| 1 | 任务 A | coder | - | 待执行 |  |
| 2 | 任务 B | tester | - | 待执行 | 可与 1 并行 |
| 3 | 任务 C | devops | - | 待执行 | 可与 1,2 并行 |
| 4 | 汇总结果 | reviewer | 1,2,3 | 待执行 | 依赖全部完成 |

## 依赖关系可视化

```
串行:
Task 1 → Task 2 → Task 3

并行:
Task 1
Task 2  → Result
Task 3

混合:
Task 1 → Task 2
      ↘ Task 3 → Task 4
```
